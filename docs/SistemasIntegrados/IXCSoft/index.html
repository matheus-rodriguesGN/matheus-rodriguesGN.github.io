IXC Soft

O IXC Soft é um sistema de gestão de provedores de internet que integra todos os dados e procedimentos administrativos, financeiros e comerciais da organização, simplificando as atividades operacionais e estratégicas possibilitando pleno controle das operações por permitir o cruzamento de dados de vários departamentos.


Veja como utilizar a Gerencianet no IXC Soft#

Em seu IXC Soft, acesse Menu Principal > Cadastros > Financeiros > Carteira de cobrança e clique em Novo;

Agora, na aba Carteira de cobrança, será configurada a conta e, por isso, no campo Conta direcione o cursor no campo e pressione o botão F2 de seu teclado ou clique na lupa para buscar;

Na janela que se abre, clique em Novo.

Para a configurar a conta, é necessário preencher os seguintes campos:

  • Conta: Preencha o nome da conta nesse campo;
  • Tipo de conta: Marque a opção Caixa;
  • Data da abertura: Coloque o dia atual;
  • Saldo da abertura: Insira pelo menos R$ 0,01;
  • Planejamento analítico: É necessário criar um planejamento analítico para essa conta (vide instruções abaixo);
  • Filial padrão: Insira aqui a filial padrão da conta;
  • Permitir pagamento com saldo negativo: Escolha uma das opções, sim ou não;
  • Ativo: Mantenha a conta marcada como ativo.

Para criar o planejamento analítico, clique na lupa do campo Planejamento analítico e após isso clique em Novo. Preencha os seguintes campos:

  • Planejamento sintético - Bancos Geral: como padrão do sistema vem como ID 2;
  • Planejamento analítico: Insira o mesmo nome da conta;

  • Clicar em Salvar.

Depois disso, faça a busca pelo planejamento analítico criado, selecione o mesmo e clique no botão Inserir Registro selecionado.

Após o procedimento acima, a conta pode ser salva clicando no botão Salvar. Na aba Banco, não insira nenhuma informação. Clique em Salvar e feche essa janela.

Selecione o registro criado e clique em Inserir registro selecionado.

Agora voltamos para a Carteira de cobrança, primeiramente precisamos marcar o Tipo de recebimento como Boleto para alterar algumas informações.

Depois disso, é necessário acessar a aba Configurações e na opção Classe das funções selecionar a opção Banco Bradesco.

Depois disso é necessário acessar a aba Carteira de cobrança, e marcar o Tipo de recebimento como Gateway. Após isso defina os campos de acordo com a necessidade:

  • Descrição: Insira o nome da carteira de cobrança nessa campo;
  • Conta: Nesse campo deve ser inserida a conta que foi criada no inicio desse tutorial;
  • Tipo de recebimento: Deve ser marcada a opção Gateway;
  • Enviar e-mail ao gerar financeiro do contrato: Se marcado como NÃO, mesmo se o tipo de cobrança do contrato for "E-mail", não irá enviar por e-mail aos clientes de forma automática na geração do financeiro;
  • Validar filial na venda: Se marcado como SIM, vai validar se a filial padrão da conta relacionada com a carteira é a mesma da saída quando o usuário for finalizar a venda ou gerar o financeiro do contrato;
  • Ativo: Define se a carteira está ativa ou não;
  • Planejamento analítico: O planejamento analítico padrão para tarifas é o ID 240, se não tiver um próprio, use o padrão;
  • Data lançamento: Define a data dos lançamentos;
  • Tipo de arredondamento: Define o arredondamento do sistema no recebimento manual.

Configurando juros, multa e descontos#

As informações que vão ser inseridas nesses campos devem ser antes validadas pelo seu contador.

  • Estando em seu sistema IXC, acesse a aba Juros/Multas/Desconto e insira os valores que deseja;
NOTA

As seções Desconto e Protesto não são utilizadas pela Gerencianet.

  • Agora, vá para a aba Configurações, que contém as seguintes opções:

    • Gateway: Para o gerenciaNet deve ser selecionada a opção Gerencianet;
    • Importado gateway: Opção que deve ser marcada como SIM se for uma carteira de boletos importados de outros sistemas;
    • Nova API: Para novas carteiras, deve ser obrigatoriamente marcado como SIM;
    • Client_Id, Client_secret, Call-Back, Identificador da conta: São informações que iremos ensinar aonde buscar (no caso de informações que estão no site do Gerencianet) e montar (no caso do call-back) no próximo tópico;
    • Tipo de impressão: Se for carteira de boletos importados deve ser selecionada a opção Importação, caso contrário, Carnê ou Fatura. Essa opção não define o layout de impressão, mas sim como ele vai ser impresso, no caso dos importados o sistema vai abrir o link do boleto no Gerencianet em nova aba;
    • Adiciona observação na descrição do item;
    • Adiciona vencimento na descrição do item: Se marcado como SIM, adiciona o vencimento na descrição do produto;
    • Envia e-mail gateways: Se marcado como SIM, o sistema vai validar e mostrar um mensagem erro se o e-mail do cliente for inválido na hora da impressão.

Configurações Gerencianet#

  • Acesse sua conta da Gerencianet e, após logar, clique API no menu;

  • Agora, clique em Minhas Aplicações > Nova Aplicação e preencha o nome da aplicação no campo (o padrão é inserir o nome como IXCSoft para identificar), deixe o modo compatibilidade DESATIVADO e clique no botão Criar nova aplicação;

  • Após isso, clique na aplicação criada; nela estarão o Client ID e Client Secret que devem ser copiados para os respectivos campos na carteira de cobrança no IXC;

  • Agora que já preenchemos o Client ID e Client Secret, resta preencher o Identificador da conta e o Call-Back; o Identificador da conta pode ser encontrado clicando em Minha conta, e o Identificador da conta estará nessa página;

  • Depois de inserir Identificador da conta na carteira de cobrança, devemos montar o Call-Back. O Call-Back é composto pelo endereço IP ou domínio do servidor que o IXC está hospedado + /retorno_gateway.php?tipo=fortunus. Ou seja, se o IP for 11.11.11.11, o Call-Back será:

    • http://11.11.11.11/retorno_gateway.php?tipo=fortunus

Após inserir o Call-Back, a carteira de cobrança está configurada.

IMPORTANTE

Lembrando que o processo de geração da primeira venda, remessa, retorno e conferência deve ser feita pelo suporte do IXC para garantir que a geração e recebimentos de boletos não causem problemas futuros.
