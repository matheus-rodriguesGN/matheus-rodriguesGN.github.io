PHP

Além da nossa API ser totalmente RESTful com respostas em JSON, oferecemos a praticidade que você precisa através de nossa SDK em PHP. Ela já está preparada para fazer sua autenticação automaticamente e possui funções para emitir suas cobranças entre outas muitas funcionalidades.


Pré-requisitos#

Os seguintes pré-requisitos devem ser considerados de acordo com a versão utilizada da SDK. Recomendamos que utilize a versão 2 da SDK.

Versão da SDKStatusPacote no PackagistCompatibilidade com PHP
1.xMantidogerencianet/gerencianet-sdk-php>= 5.4 (não é compatível com PHP 7.x)
2.xMantidogerencianet/gerencianet-sdk-php>= 5.5 (compatível com PHP 7.x)

Instale nossa SDK com mais facilidade através do gerenciador de dependências Composer. Ou se preferir siga nosso passo a passo da instalação sem composer.


Instalando a SDK com composer#

Caso você não possua o Composer instalado veja os passos "Instalando o composer no windows" ou "Instalando o composer no linux"

  1. Agora que você já tem o composer em sua máquina, utilize o terminal (ou CMD do windows) e entre na pasta raiz do seu projeto para adicionarmos a dependência, conforme abaixo:
  • 2.x
  • 1.x
{"require":
{
"gerencianet/gerencianet-sdk-php": "2.*"
}
}

This is a very long line that will still be quoted properly when it wraps. Oh boy let's keep writing to make sure this is long enough to actually wrap for everyone. Oh, you can put Markdown into a blockquote.

  1. Após a execução do comando, todas as dependências serão instaladas em seu diretório, inclusive a pasta vendor com o arquivo autoload.php . Após a instalação você verificará que na raiz do projeto surgiu um arquivo composer.json . Ele é o arquivo responsável por gerenciar os pacotes do composer que são usados pelo seu projeto. Ele terá uma estrutura parecida com a abaixo:
{"require":
{
"gerencianet/gerencianet-sdk-php": "2.*"
}
}

Pronto, agora é só começar a utilizar as soluções de integração da Gerencianet. Veja a tabela com os todos os tipos de integrações disponíveis.

Cabe frisar que caso você necessite excluir o pacote, basta entrar novamente na raiz do seu projeto e digitar o comando abaixo:

// Este é um exemplo de pasta de projeto utilizando wamp no windows.
$ cd \wamp\www\meu-projeto
$ composer remove gerencianet/gerencianet-sdk-php

Instalando a SDK sem composer#

O uso do Composer (gerenciador de dependências) é recomendável, mas não obrigatório. Caso seja de seu interesse prosseguir sem utilizá-lo, você pode baixar diretamente em nosso repositório através das branches de distribuição para diretório do seu projeto.

Atualmente, oferecemos duas versões, intituladas 2.x(master) e 1.x , sendo:

  • 2.x - master (Recomendada): Ela é a nossa versão mais atual, utilizando o Guzzle (^6.0.0) e é compatível com as versões mais recentes do PHP 7.1, 7.0, 5.6 e 5.5). Baixar agora
  • 1.x : Versão mais antiga da SDK e mantida por usar Guzzle(5.3.0) e compatível com o PHP 5.4. Baixar agora

A menos que você tenha um sistema incompatível com PHP 5.5 ou posterior, recomendamos que siga com a instalação padrão.

Importante reforçar que o conteúdo presente esse .zip é apenas um requisito para que você possa começar a utilizar a SDK em PHP da Gerencianet. Esta pasta por si só não é um "exemplo pronto" de uso da API Gerencianet, mas a SDK em PHP da Gerencianet que permite a utilização da API.


Instalando o Composer no Windows#

Caso você já possua o Composer instalado você pode pular para o passo"Instalando a SDK com composer"

  1. Primeiramente, vamos baixar o instalador do Composer para Windows neste link e, assim que o download for finalizado, execute-o;
  2. Você precisará informar o caminho de instalação do seu PHP. Caso você esteja utilizando um servidor Wamp, por exemplo, e na instalação surgir uma mensagem relacionada ao arquivo "openssl", você precisará informar seu arquivo *.exe do PHP contido no diretório raiz do Wamp (geralmente em \wamp\bin\php\php.5.6.16). Cabe frisar que a pasta php.5.6.16 refere-se a versão do PHP que está sendo utilizado na confecção dessas instruções, portanto, esse diretório poderá estar com outro nome, conforme a versão do seu PHP;
  3. Durante a instalação, após indicar o diretório do seu PHP, clique em next. A instalação poderá retornar uma mensagem de alerta relacionado à configuração do "openssl". Trata-se de uma mensagem comum que significa que o "openssl" está desabilitado, contudo, vamos resolver de forma rápida e fácil.

Caso haja alguma mensagem sobre o openssl, prossiga para o "Instalando a SDK com composer"

Para habilitar seu "openssl", será necessário alterar o arquivo php.ini.

  1. Abra o diretório de instalação do seu php e localize o arquivo php.ini;

  2. Abra o arquivo php.ini em um editor de texto (p. ex: notepad++, sublime, etc) e pressione CTRL + F e pesquise pela palavra “openssl” (sem as aspas);

  3. O sinal de ponto e vírgula ( ; ) desabilita o arquivo php_openssl.dll. Apague este sinal e ele habilitará o arquivo, ou seja:

    Está assim: extension=php_openssl.dll

    Deve ficar assim: extension=php_openssl.dll

    Agora, salve o documento (pressione CTRL + S ) e feche o arquivo.

  4. A extensão estará habilitada e você poderá continuar com a instalação. Para que a instalação seja atualizada em relação ao procedimento, é importante que você retorne uma tela e depois avance com a instalação normalmente.
